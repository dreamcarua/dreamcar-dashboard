3. **Нет отдельной удобной формы** добавления любой расходной операции прямо в финансовом модуле.

4. **На дашборде** витрати показаны одной цифрой — не видно структуры расходов по группам.

Дать Вадиму возможность **внести любую трату** проекта (от покупки BMW за 3 млн до оплаты канцтоваров за 500 грн) в **одном месте**, без необходимости указывать UTM-метки. Убрать зависимость от Excel таблиц — всё в финансовом модуле.

### ЧТО на выходе

1. **Расширенный справочник категорий** — 5 групп (Реклама, Виробництво, Операційні, Команда, Інше) с 25+ конкретными категориями

## ⚠️ EDGE CASES И РИСКИ

### Потенциальные проблемы

1. **Обратная совместимость категорий**
   - Текущие расходы в БД имеют category: `meta`, `google`, `viber`, `sms`, `email`, `design`, `other`
   - После изменения структуры константы — эти значения должны ОСТАТЬСЯ валидными
   - **Решение:** `getFlatCategories()` возвращает все ключи из всех групп, существующие категории остаются в группе "advertising"

2. **Проект в manual_costs vs finance_projects**
   - В manual_costs поле "Проект" содержит короткие имена: BMW, VOLVO, MERCEDES
   - В finance_projects имена полные: BMW X5 HYBRID, VOLVO XC90, BMW 330E HYBRID
   - **Решение:** `FinanceTransaction::resolveProjectId()` уже использует alias map — работает

3. **Новая форма "Додати витрату" должна использовать finance_project_id напрямую**
   - В отличие от manual_costs (там имя проекта через alias), новая форма должна показывать список из `projects.list` API
   - Передавать `project_id` как integer, НЕ через alias

4. **Права доступа (canWrite)**
   - Кнопка "Додати витрату" должна быть видна только admin и finance_manager
   - Backend action должен проверять `FinanceAuth::canWrite()` и возвращать 403 если нет прав

5. **Карта в новой форме**
   - Если выбрана карта но её баланс меньше суммы → warning, но разрешить (карта может быть с кредитным лимитом)
   - `deductBalance()` уже защищён `GREATEST(0, balance - amount)` — в минус не уйдёт

6. **Большие суммы (покупка авто 3 млн)**
   - `amount_uah DECIMAL(15,2)` — поддерживает до 9 999 999 999 999.99 ₴ — более чем достаточно
   - JS `parseFloat()` справится с такими числами

7. **Валюта**
   - В новой форме НЕ добавляем выбор валюты — всё в UAH
   - Если клиент купит машину в USD — конвертация вручную перед вводом

8. **Рендер модалки на мобильном**
   - CSS должен учитывать узкий экран
   - На ширине <768px — форма одной колонкой, модалка во весь экран

### Обратная совместимость

- ✅ Существующие расходы не ломаются (категории остаются в группе "advertising")
- ✅ manual_costs.php продолжает работать для маркетологов как раньше

          $newBalance = $card ? (float)$card['balance_uah'] : null;
